# Flood Forecast — Kacha River Flood Prediction System

Система прогнозирования уровня воды и оценки паводковой ситуации для **реки Кача (Красноярск)**.

Проект разработан в рамках инженерного хакатона и объединяет:

- сбор и обработку данных гидрологических сенсоров,
- построение и обучение моделей машинного обучения,
- математическую модель оценки риска,
- веб-приложение для мониторинга и визуализации паводковой ситуации.

Система позволяет анализировать текущую гидрологическую ситуацию, прогнозировать изменение уровня воды и оценивать риск паводков на горизонте **6 / 24 / 72 часов**.

---

# Постановка задачи

Цель проекта — разработать алгоритм прогнозирования паводковой ситуации на основе исторических данных уровня воды.

Исходные данные представляют собой временной ряд измерений уровня воды, полученных с сенсоров, установленных на мостах через реку Кача.

Каждое измерение содержит:

- timestamp — время измерения  
- bridge_id — идентификатор моста / сенсора  
- water_level — расстояние от датчика до поверхности воды  

Особенности данных:

- временной ряд содержит пропуски и шум  
- измерения могут быть неравномерными  
- данные представлены только за **1 год наблюдений**

Задача решается в гибридной постановке (**вариант C**):

1. **Регрессия** — прогноз уровня воды на горизонте времени  
2. **Классификация риска** — определение уровня паводковой опасности

Прогноз строится на горизонтах:

- **+6 часов**
- **+24 часа**
- **+72 часа**

# Данные

Основной датасет формируется из нескольких источников:

## Данные сенсоров

Сенсоры установлены на мостах через реку Кача и измеряют расстояние до поверхности воды.

Основные поля:

| Поле | Описание |
|-----|-----|
| bridge_id | идентификатор сенсора |
| timestamp | время измерения |
| water_level | расстояние от сенсора до поверхности воды |

Интерпретация:

чем **меньше значение water_level**, тем **выше уровень воды**.

---

## Погодные данные

Для повышения качества прогноза используются погодные данные:

| Параметр | Описание |
|------|------|
| temperature | температура воздуха |
| precipitation | количество осадков |
| humidity | влажность |
| pressure | атмосферное давление |

Погодные данные получены через **Open-Meteo API**.

# Предобработка данных

Перед обучением модели данные проходят этап подготовки.

Pipeline выполняет следующие шаги:

1. очистка исходных данных сенсоров  
2. приведение временных меток к единому формату  
3. агрегация измерений до почасового ряда  
4. объединение с погодными данными  
5. генерация признаков временного ряда  

Создаются дополнительные признаки:

### Лаговые признаки

- water_level_lag_1h  
- water_level_lag_3h  
- water_level_lag_6h  
- water_level_lag_12h  
- water_level_lag_24h  

### Производные признаки

- изменение уровня воды  
- скорость изменения уровня  

### Скользящие статистики

- среднее значение уровня воды  
- стандартное отклонение  
- максимальное значение за окно времени  

### Погодные признаки

- накопленные осадки за 6 / 12 / 24 / 72 часа  
- средняя температура  
- изменение температуры  

### Календарные признаки

- час  
- день недели  
- месяц  
- день года

# Веб-приложение

Интерфейс системы реализован в виде веб-приложения, которое позволяет визуализировать данные и прогнозы.

Основные возможности:

- интерактивная карта реки Кача  
- отображение мостов с сенсорами  
- просмотр текущего уровня воды  
- прогноз уровня воды на 6 / 24 / 72 часа  
- индекс паводковой опасности  
- визуализация зон потенциального разлива  
- система уведомлений  

Приложение также поддерживает:

- выбор даты и времени  
- просмотр исторических данных  
- адаптивный интерфейс  
- тёмную и светлую тему
