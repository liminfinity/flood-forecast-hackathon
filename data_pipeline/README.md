# Flood Forecast — Data Pipeline

Проект предназначен для подготовки датасета для модели прогнозирования уровня воды и оценки риска паводков.  
Пайплайн очищает исходные данные датчиков, добавляет погодные данные и строит признаки временного ряда, которые можно использовать для обучения моделей машинного обучения.

---

# Назначение проекта

Задача проекта — разработать алгоритм прогнозирования уровня воды и риска паводка на основе исторических временных рядов наблюдений у мостов.

Решение включает:

- анализ и очистку исходных данных
- агрегацию временного ряда
- интеграцию внешних погодных данных
- построение признаков
- формирование итогового датасета для обучения моделей

---

# Архитектура пайплайна

Пайплайн состоит из нескольких этапов:

1. очистка исходного датасета
2. преобразование временных меток
3. агрегация измерений уровня воды
4. загрузка погодных данных
5. генерация признаков временного ряда
6. объединение данных в единый датасет

# Входные данные

Исходный датасет содержит измерения уровня воды с датчиков.

Пример исходных полей:

- `id` — идентификатор записи
- `sensor_id` — идентификатор датчика / моста
- `value` — измеренный уровень воды
- `ts_create` — время измерения (Unix timestamp)
- `ts_update` — время обновления записи

После очистки данные приводятся к стандартному виду:

- `bridge_id`
- `water_level`
- `timestamp`

---

# Внешние данные

Для улучшения качества модели используются погодные данные.

Источником является **Open-Meteo API**.

Используемые погодные параметры:

- температура воздуха
- влажность воздуха
- атмосферное давление
- осадки
- дождь

Эти данные позволяют учитывать влияние погодных условий на уровень воды.

---

# Файл координат мостов

Для загрузки погодных данных используется файл:

```
config/bridges.csv
```

Пример содержимого:

```csv
bridge_id,latitude,longitude,name
1,56.0104,92.8522,Bridge 1
2,56.0184,92.8671,Bridge 2
3,56.0250,92.8800,Bridge 3
```

Поля:

- `bridge_id` — идентификатор поста наблюдения
- `latitude` — широта расположения моста
- `longitude` — долгота расположения моста
- `name` — человекочитаемое название моста

# Выходные файлы

После запуска пайплайна создаются следующие файлы.

## base_clean.csv

Очищенный исходный датасет.

Поля:

| Поле | Описание |
|-----|----------|
| `bridge_id` | идентификатор моста или поста наблюдения |
| `water_level` | измеренный уровень воды относительно датчика |
| `timestamp` | время измерения уровня воды |

---

## weather_openmeteo.csv

Погодные данные, загруженные из Open-Meteo.

| Поле | Описание |
|-----|----------|
| `bridge_id` | идентификатор моста, для которого получены погодные данные |
| `timestamp` | время погодного измерения |
| `temperature_2m` | температура воздуха на высоте 2 метров |
| `relative_humidity_2m` | относительная влажность воздуха |
| `surface_pressure` | атмосферное давление на поверхности |
| `precipitation` | количество осадков за последний час |
| `rain` | количество дождевых осадков за час |

---

## dataset_with_external_features.csv

Итоговый датасет для обучения модели.

Этот файл содержит объединённые данные уровня воды, погоды и построенные признаки.

# Итоговые признаки

## Базовые

| Поле | Описание |
|-----|----------|
| `bridge_id` | идентификатор моста или гидропоста |
| `timestamp` | временная метка наблюдения |
| `water_level` | фактический уровень воды в данный момент времени |
| `measurements_in_hour` | количество исходных измерений, попавших в данный час |

---

## Погодные признаки

| Поле | Описание |
|-----|----------|
| `temperature_2m` | температура воздуха |

| `temperature_24h_mean` | средняя температура за последние сутки |

| `precipitation_24h` | суммарные осадки за последние 24 часа |

# Установка зависимостей

Создать виртуальное окружение:

```
python -m venv venv
source venv/bin/activate
```

Установить зависимости:

```
pip install -r requirements.txt
```

---

# Запуск пайплайна

```
python src/build_dataset.py \
  --raw data/raw_dataset.csv \
  --bridges config/bridges.csv \
  --out-dir data/processed
```

---

# Запуск через Docker

## Сборка образа

```
docker build -t flood-data-pipeline .
```

## Запуск контейнера

```
docker run --rm \
  -v $(pwd)/config:/app/config \
  -v $(pwd)/data:/app/data \
  flood-data-pipeline \
  --raw /app/data/raw_dataset.csv \
  --bridges /app/config/bridges.csv \
  --out-dir /app/data/processed
```

# Ограничения текущей версии

Текущий пайплайн:

- не строит таргеты прогноза (6h, 24h)
- не учитывает upstream-гидропосты
- не включает снежный покров
- не включает влажность почвы
- не обучает модель автоматически
